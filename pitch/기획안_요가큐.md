> **용어 정리** — `요가큐` = 사용자에게 노출되는 서비스/브랜드 이름. `elbee` = 챗봇 페르소나 이름(말하는 캐릭터). `aeogeo` = 백엔드 코드네임/Repo. 본 기획안에서 "요가큐"는 서비스 전반을, "elbee"는 챗봇 인터페이스를 가리킨다.
>
> **구현 전제** — 본 기획안의 페이지 인벤토리·챗봇 UX·API 연결은 **신규 구현 대상 스펙**이다. 기존 `aiegoo/yoga` 레포는 도서 OCR / 학습자료 / 레퍼런스 자산 저장소로만 사용하며, 사용자/스튜디오 운영자가 만나는 챗봇 UI는 본 기획안 스펙에 맞춰 새로 만든다.

# **📍** 서비스 제작 목적

요가큐의 핵심 목적은 **요가 수련자가 자신의 몸·일정·날씨에 맞는 수련을 찾도록 돕고, 동시에 그 수련 데이터가 AI 어시스턴트(ChatGPT, Perplexity, Naver Cue 등)에서 인용 가능한 공식 자산이 되게 만드는 것**이다.

기존 요가 앱·검색 서비스는 대체로 “영상을 추천하는 곳”, “스튜디오 목록을 보여주는 곳”에 머무른다.
하지만 실제 요가 수련에서는 단순 추천보다 더 중요한 요소들이 있다.

* 이 사람의 몸에 **금기(허리 디스크, 고혈압, 임신 등)** 가 있는지
* 지금 **몇 분이** 가능한지, **어디에** 있는지
* **비가 오면** 야외 클래스를 피해야 하는지
* 추천된 수업이 **왜** 추천되었는지 — 근거가 보이는지
* 그 답이 **AI 어시스턴트에 그대로 인용**되는지

요가큐는 이 문제를 해결하기 위해 만들어진다.

1. **수련자(member)에게는** — 몸의 제약과 일정·날씨를 고려한 안전하고 구체적인 수업/스튜디오를 한국어 해요체 챗봇 `elbee`로 안내한다.
2. **스튜디오 운영자(provider)에게는** — 주변 수련자에게 발견되는 채널을 제공하고, **운영자 전용 챗봇 `elbee Studio`** 가 슬롯 운영·날씨 라우팅·후기 응답을 대화형으로 도와준다.
3. **AI 어시스턴트 사용자에게는** — JSON-LD / 구조화된 답변으로 요가큐의 데이터를 직접 인용 가능하게 만들어, 검색 트래픽 없이도 발견되게 한다.
4. **모두에게** — 단발성 검색이 아니라, 수련 이력과 피드백이 다음 추천을 더 정확하게 만드는 **성장형 컨텍스트 구조**를 제공한다.

결국 요가큐의 최종목표는
**“좋은 몸 상태가 좋은 수련을 만나고, 그 경험이 다시 더 정밀한 추천과 신뢰가 되어 선순환을 만드는 한국어 우선 요가 큐레이터”** 의 제공이다.

# **🔑 플랫폼 방향 정의**

elbee는 일반적인 “요가 추천 앱”이 아니라, 다음과 같이 정의한다.

**한국어 네이티브 RAG로 운영되며, AI 어시스턴트에 인용되는 요가 GEO 큐레이터 (Generative Engine Optimization).**

* 단순 영상 랭킹이 아니라 **금기·근접·날씨**를 동시에 고려하는 매칭
* OpenAI 등 외부 LLM 의존성 없이 **로컬 우선(Ollama)** 으로 동작 — PII / 헬스 데이터 외부 송신 없음
* 165쪽 GEO 도서 OCR + 28k 단어 색인을 **공식 도메인 지식**으로 갖춤
* 결과는 **JSON-LD + cite-able 스니펫**으로 노출되어 ChatGPT·Perplexity·Naver Cue가 그대로 인용
* 한국어 해요체 + 받침/조사 자동 결정으로 **브랜드 보이스가 깨지지 않음**

# **📚 사이트 설명과 장점**

“좋은 컨텍스트가 좋은 수련을 만든다”는 슬로건으로,

## 한국어 우선 RAG × GEO 큐레이션으로 작동하는 안전한 요가 매칭 플랫폼

### **1. 컨텍스트 중심 매칭 (Context-Centric Matching)**

elbee는 단순한 키워드 검색을 넘어, **금기(contraindications) · 위치 · 날씨 · 어메니티 · 시간**이 유기적으로 결합된 컨텍스트를 매칭의 1차 입력으로 사용한다.

* **컨텍스트 입력 요소:**
  * 사용자의 신체 제약(허리 디스크, 고혈압, 임신, 무릎 등)
  * 위경도(geo_coord) — Haversine 근접도
  * 가용 시간(분)
  * 실시간 날씨 (rain → weather_indoor=true 자동 필터)
  * 선호 어메니티 (샤워, 매트 대여, 락커, 주차)

### **2. 한국어 네이티브 응답 (Korean-Native Voice)**

답변은 영어 번역체가 아니라, **해요체** + 받침/조사 자동 결정으로 자연스러운 한국어로 생성된다.

* `templates.py` 의 `josa()` 가 받침을 보고 은/는, 이/가, 을/를 자동 선택
* `ELBEE_SYSTEM_KO` 시스템 프롬프트가 톤앤매너를 고정
* few-shot 예시로 어투(invite / search / chat) 분리
* 도메인 신조어(요린이·회복요가·인요가) 야간 재인덱싱

### **3. 로컬 우선 RAG (Local-First, Zero PII Leak)**

LLM은 **Ollama (llama3) 로컬 런타임**으로 동작한다. 호출당 비용이 0이며, 헬스 컨텍스트가 외부로 나가지 않는다.

* `httpx` 비동기 → `Ollama :11434`
* `/chat/stream` 으로 SSE 점진 응답
* `llama3 → HyperCLOVA-X` 무중단 교체 가능

### **4. AEO + GEO 듀얼 노출 (Cited by AI)**

elbee의 결과는 사람이 보는 UI **그리고** AI 어시스턴트가 가져가는 JSON-LD 두 채널로 동시 노출된다.

* `LocationSearchResponse` 가 그대로 schema.org 호환 구조
* `invite_message` 는 brand_logo + 주소 + 좌표를 동반 — 그대로 인용 가능
* `sources[].page` 가 165쪽 도서의 페이지를 직접 가리켜 reference 트레이서빌리티 확보
* 결과: ChatGPT/Perplexity/Naver Cue 답변에 elbee 데이터가 원형 그대로 삽입

### **5. 설명 가능한 추천 (Explainable Match)**

단순 점수가 아니라 **왜 이 스튜디오/수업이 추천되었는지**를 명시한다.

* 매칭 근거 예시:
  * "디스크 → 후굴(backbend) 자동 제외"
  * "현재 비 → 실내 클래스 자동 우선"
  * "현재 위치에서 0.3km — 도보 4분"
  * "재방문 의사 92% / 평점 4.8"
  * "샤워 + 매트 대여 보유"

### **6. 양방향 신뢰 자산 (Bi-directional Trust)**

수련자만 평가받는 구조가 아니라 스튜디오도 **응답성·정확성·실측치**로 평가받는 구조.

* 스튜디오 평가 항목: 슬롯 정확도, 날씨 라우팅 적용률, 후기 응답률, 사진 최신성
* 수련자 평가 항목: 출석률, 후기 작성 비율, 강사 피드백 친화도

### **7. 성장형 컨텍스트 (Compounding Context)**

수련 이력이 쌓일수록 추천 정확도가 자연스럽게 올라간다.

* 선순환 고리: `수련 이력 누적` → `금기·선호 학습` → `더 정밀한 매칭` → `만족도 상승` → `재방문` → `새 수련 데이터`

# **⭐ 챗봇 / 네비게이션**

## **9-1. AI 인사이트 검색 (정보 제공형 AI)**

메인 홈 최상단에 배치되어 로그인/비로그인 사용자 모두에게 165쪽 GEO 도서 + 8개 시드 스튜디오 데이터를 기반으로 답변한다.

* **핵심 기능:**
  * **수업 추천 with 금기 필터:** "허리 디스크 / 15분"을 입력하면 후굴·점프 빠진 추천만 반환.
  * **근접 스튜디오 매칭:** 위경도 + radius_km 입력 시 Haversine + 날씨 필터 적용 결과.
  * **출처 페이지 제시:** `sources[]` 로 도서 페이지 직접 인용.

## **9-2. ElbeeBot (수련 안내 챗봇) — `POST /chat`**

수련자가 자유 텍스트로 질문하면 RAG로 답변. SSE 스트리밍 지원.

* **기능:**
  * **AI 응답:** ELBEE_SYSTEM_KO + 165쪽 도서 RAG로 해요체 답변.
  * **위치 기반 초대:** geo_coord 동봉 시 자동으로 근접 스튜디오 invite_message 추가.
  * **체크 가이드:** 첫 수련 / 임산부 / 부상 회복 등 분기별 안전 가이드.

## **9-3. 네비게이션 고도화**

### **(수련자) 컨텍스트 중심 탐색**

* **내 수련 브리프:** 비정형 자가 진단(허리 안 좋아요 + 지금 비 와요 + 30분 가능)을 구조화된 매칭 입력으로 자동 변환.
* **스마트 스튜디오 매칭:** 후보 스튜디오를 거리·실내가능·어메니티로 한 화면에서 비교.
* **신뢰 지표 관리:** 스튜디오의 응답률, 평점, 슬롯 정확도 확인.

### **(스튜디오) 운영 중심 탐색**

* **슬롯 관리:** 시간·인원·실내외 토글.
* **날씨 라우팅 검토:** 비 예보 시 실내 슬롯이 자동 우선되는 영향 미리보기.
* **후기 응답:** 수련자 후기에 해요체로 답변.

# **🤱 사용자 타겟층**

### **1) 메인 타겟**

* 허리 디스크·임신·고혈압 등 **금기 조건이 있는** 수련자 (안전한 수련을 찾기 어려움)
* 자가 진단 + 위치 + 날씨를 한 번에 답해주는 **스마트 어시스턴트형 검색**을 원하는 수련자
* AI 어시스턴트(ChatGPT/Perplexity)로 운동 정보를 검색하는 **검색 행동이 변한 신세대**
* 한국어 해요체로 자연스러운 안내를 받고 싶은 **한국 거주 수련자**
* 주변 스튜디오의 슬롯/어메니티/날씨 라우팅까지 한 번에 보고 싶은 **출퇴근형 직장인**

### **2) 서브 타겟**

* 운영 채널이 부족해 발견되지 않는 **소규모 스튜디오**
* 비 오는 날 실내 클래스를 자동으로 우선 노출하고 싶은 **다점포 스튜디오**
* AI 어시스턴트에 자기 데이터가 인용되길 원하는 **요가 콘텐츠 크리에이터**
* 165쪽 GEO 도서 + 도메인 지식으로 **레퍼런스가 정확한** 답변을 원하는 강사·연구자

# **🔔 핵심 기능 구성**

### **1) AI 자가 진단 채팅 — `POST /chat`**

자유 텍스트(허리가 안 좋고, 지금 비 오고, 30분 있어요)를 받아 RAG로 안전한 수업/스튜디오를 추천. SSE 스트리밍 (`/chat/stream`).

응답 형식:
* answer (해요체)
* sources[] (도서 페이지 인용)
* brand_logo
* language="ko"

### **2) 위치/날씨/어메니티 매칭 — `POST /search/locations`**

geo_coord + weather + amenities + tags 를 받아 Haversine 근접 + 날씨 라우팅 + 어메니티 필터를 한 번에 적용.

응답 형식:
* results[] (id, name, district, lat, lng, distance_km, weather_indoor, rating, **invite_message** 해요체)
* total / message / brand / brand_logo

### **3) 키워드 + 토픽 검색 — `POST /search` 외**

165쪽 GEO 도서 색인(page · keyword · platform · region · topic)을 키워드/필터로 조회.

* `/search/suggest` — 프리픽스 자동완성
* `/search/filters` — 사용 가능한 platform/region/topic 목록
* `/search/health` — 인덱스 크기 / 준비 상태

### **4) 페르소나 엔진 (해요체 + 조사 자동 결정)**

`templates.py`:
* `josa(word, has_batchim, no_batchim)` — 받침으로 은/는, 이/가, 을/를 자동 선택
* `ELBEE_SYSTEM_KO` — 시스템 프롬프트 (해요체 고정)

### **5) OCR 인테이크 파이프라인 (오프라인)**

* `screenshots/*.png` → OpenCV 전처리 → pytesseract → 정제 + 청킹 + 토픽 태깅
* 산출: `ocr_database.json` (165 페이지 · 약 28k 단어), `page_index.json`, `keyword_index.json`
* `scripts/adapt_content.py` → Jekyll Markdown 변환
* `scripts/integrate.py` → yoga-chatbot/references/ 동기화
* `watchdog` 기반 자동 OCR

### **6) JSON-LD / AEO 게시 레이어**

ChatGPT, Perplexity, Naver Cue 등이 elbee 데이터를 답변에 그대로 인용할 수 있도록 schema.org 호환 구조화 데이터 게시.

### **7) 헬스/모니터링**

* `/health`, `/chat/health`, `/search/health` 다층 헬스 체크
* lifespan 훅에서 GeoDataStore + YogaLocationStore 워밍

# **🛠 사용 기술**

이 기술 스택은 **'한국어 네이티브 + 로컬 우선 + AI 인용 가능 + GEO 컨텍스트 매칭'** 이라는 목표를 달성하기 위해 설계되었다. (근거: aeogeo issues #1 SSOT, #4 한국어 NLP/GEO).

### **○ 한국어 NLP 자연성 전략 (issue #4 A·B·C 반영)**

# 🎬 스토리보드 (User Flow)

## **🎬 핵심 스토리보드 (A. User Flow / B. 인터페이스 관점)**

자가 진단 → 안전 필터 → 컨텍스트 매칭 → 신뢰 형성 → 재방문 → 더 정밀한 추천

## **🧐 관점별 시나리오 요약**

### **관점 1. 비로그인 방문자**
홈 진입 → "AI에 인용되는 데이터" 메시지 확인 → 샘플 질문 시도 → 추천 카드 → 회원가입 유도

### **관점 2. 수련자(member)**
회원가입 → 자가 진단(금기·시간·위치) → AI 채팅 추천 → 스튜디오 초대 메시지 → 예약 → 수련 → 후기 → 재방문 추천 정밀화

### **관점 3. 스튜디오 운영자**
회원가입 → 슬롯/어메니티/실내외 등록 → 날씨 라우팅 미리보기 → 매칭 노출 → 후기 응답 → 신뢰 지표 상승

### **관점 4. AI 어시스턴트 사용자**
ChatGPT/Perplexity에서 "강남 비 오는 날 30분 요가" 검색 → elbee의 JSON-LD 응답이 그대로 인용 → 클릭 유입 → 회원가입

## **🎞️ 핵심 EP별 스토리보드**

### **Ep 1. 첫 방문 → "여기는 영상 추천 앱이 아니라 RAG 큐레이터구나"**

#### **A. User Flow**
* 사용자는 elbee.yogaman.club 에 들어온다.
* "또 운동 영상 사이트인가?" 라고 처음엔 생각한다.
* 메인 화면을 보면 다르다. 영상 썸네일이 아니라 "허리 안 좋고, 지금 비 오고, 30분 가능해요" 같은 자가 진단 입력창이 중심이다.
* 답이 영상이 아니라 **근처 실내 스튜디오 + 도서 페이지 인용 + 해요체 안내**로 돌아온다.
* "ChatGPT처럼 답하는데 출처가 분명하고, 한국어가 자연스럽고, 위치까지 알아주네" 라고 인지한다.
* 비로그인 상태에서도 샘플 답변을 둘러보면서 이 서비스의 핵심이 **'영상 랭킹'이 아니라 '컨텍스트 매칭'** 임을 이해한다.
* 관심이 생기면 자기 위치/금기를 저장하고 싶어한다 → 회원가입 유도.

#### **B. 인터페이스 관점**
* 메인 홈 최상단: 자가 진단 입력창 (영상 썸네일 X)
* 메시지: "좋은 컨텍스트가 좋은 수련을 만든다"
* 역할별 CTA: '수련 시작 / 스튜디오 등록 / 샘플 답변 보기'
* 비로그인 상태에서도 샘플 8개 스튜디오 카드 + 165쪽 도서 토픽 태그는 노출
* 저장 / 예약 / 후기 작성을 누르면 회원가입 모달

---

### **Ep 2. 수련자 첫 가입 → "내 몸 상태를 한 번만 입력하면 다음부터 자동 반영되네"**

#### **A. User Flow**
* 가입 직후 빈 화면이 아니라 **자가 진단 온보딩**이 뜬다.
* 단계: 금기 선택(허리 디스크 / 임신 / 고혈압 / 무릎 / 어깨 / 없음) → 선호 시간대 → 선호 어메니티 → 위치 권한.
* "한 번만 입력하면 다음 검색부터 자동으로 적용돼요"라는 메시지가 보인다.
* 사용자는 "내가 매번 '디스크예요' 라고 말하지 않아도 되겠네" 라고 느낀다.

#### **B. 인터페이스 관점**
* 회원가입 직후 빈 대시보드 대신 **자가 진단 마법사** 노출
* 진행률 바 + "70% 입력 시 추천 정확도 +30%" 메시지
* 마지막 단계에서 위치 권한(geolocation API) 요청
* 저장 후 "첫 추천 받기" CTA → `/chat` 자동 호출

---

### **Ep 3. 수련 추천 → "왜 이 수업이 추천되었는지가 보인다"**

#### **A. User Flow**
* 수련자는 "오늘 30분, 허리만 풀고 싶어요" 같이 자유롭게 입력.
* AI 답변이 해요체로 돌아온다.
* 답변 아래 **추천 근거 칩**이 보인다: `디스크 → 후굴 제외` `현재 비 → 실내` `0.3km · 도보 4분` `평점 4.8`
* `sources[]` 에 165쪽 도서의 페이지 번호가 직접 링크되어 있다.
* 사용자는 "이건 GPT가 그냥 답한 게 아니라 진짜 출처가 있구나" 라고 느낀다.

#### **B. 인터페이스 관점**
* 답변 카드: 본문 + 근거 칩 + 출처 페이지 + invite_message
* invite_message 는 별도 카드로 강조 (스튜디오 이름 / 거리 / 슬롯 / 해요체 초대문)
* "다른 옵션 보기" 버튼으로 결과 목록 전환
* 답변 우측에 "이 답변 인용하기" → JSON-LD 복사 가능

---

### **Ep 4. 위치 기반 초대 → "비 오니까 알아서 실내로 바꿔주네"**

#### **A. User Flow**
* 사용자는 위치를 켜고 "지금 갈 수 있는 데" 라고 묻는다.
* 시스템이 weather=rain 을 자동 감지 → weather_indoor=true 만 필터.
* 결과 1순위: 성수 elbee 요가 스튜디오 (0.3km, 실내, 매트 대여, 평점 4.8).
* invite_message: "안녕하세요, elbee 멤버! 🙏 오늘 성수동은 근처 성수 elbee 요가 스튜디오에서 수련하실 기회가 생겼어요. 우천 시 실내 수업으로 자동 매칭됩니다."
* 사용자는 "내가 비 온다고 말 안 했는데 알아서 챙겨주네" 를 느낀다.

#### **B. 인터페이스 관점**
* 결과 카드: 거리 / 어메니티 칩 / 실내·실외 라벨 / 평점 / 후기 N건
* "지도에서 보기" → Kakao Map 임베드
* "예약 문의" → 스튜디오 채팅 (또는 전화/Kakao 채널 연결)
* 우측 사이드: "왜 추천?" 패널 펼침 → 매칭 근거 전부 노출

---

### **Ep 5. 스튜디오 등록 → "비 오는 날 자동 우선 노출이 정말 되네"**

#### **A. User Flow**
* 스튜디오 운영자는 가입 후 슬롯 / 실내외 / 어메니티 / 사진을 등록.
* 등록 완료 시 **날씨 라우팅 미리보기** 가 뜬다 — 다음 7일 예보 기준으로 본 매장이 몇 회 우선 노출되는지 시뮬레이션.
* 운영자는 "비 예보일에 내 매장이 자동으로 위로 가네" 라고 인지.

#### **B. 인터페이스 관점**
* 슬롯 등록: 시간 / 인원 / 강사 / 실내외 토글 / 어메니티 체크
* 미리보기: 7일 캘린더에 우선 노출 횟수 + 예상 노출 수 그래프
* "공개" 토글로 즉시 매칭 풀에 진입

---

### **Ep 6. 매칭 후 후기 → "후기가 다음 추천에 진짜 반영된다"**

#### **A. User Flow**
* 수련 종료 후 elbee 가 자동으로 짧은 후기 모달을 띄운다 (강도 / 정확도 / 안전성 / 재방문 의사 / 한 줄).
* 후기를 남기면 "이 후기는 다음 추천에 반영돼요" 라는 안내가 뜬다.
* 다음 검색에서 사용자에게 더 적합한 강도가 자동으로 우선된다.

#### **B. 인터페이스 관점**
* 후기 모달: 슬라이더 4개 + 1줄 텍스트 + 사진 첨부 (선택)
* 제출 후 화면: "다음 추천이 +14% 더 정확해졌어요" 게이지
* 스튜디오 측: 후기 응답 카드 → 해요체 답글 작성 도우미

---

### **Ep 7. AI 어시스턴트 인용 → "ChatGPT가 elbee 데이터를 그대로 가져갔다"**

#### **A. User Flow**
* 외부 사용자가 ChatGPT에서 "강남 비 오는 날 30분 요가" 라고 검색.
* ChatGPT 응답에 elbee 의 JSON-LD 가 그대로 인용되어 있다.
* 사용자가 그 링크를 클릭 → elbee 사이트 유입 → Ep 1 으로 진입.
